agentify run

Pick a provider loadout.

agentify run launches Codex, Claude, Gemini, OpenCode, or a custom command from an Agentify-prepared repository. Choose a fresh task, an interactive shell, a resumed provider thread, or a managed-context route before the provider starts.

choose mode command index
default fresh task resume only with --continue or --resume context modes compact · routed
quick launch provider ready 
# Fresh provider task with compact prompt
agentify run --provider codex "add checkout retry tests"

# Open the provider UI from this repo
agentify run --provider codex

# Resume the provider's own latest conversation
agentify run --provider codex --resume
mental model

Run is a launch screen, not a save file.

Use run when one provider launch should handle the work. It can carry Agentify-selected context into the prompt, but durable Agentify session state belongs to agentify sess.

new game

agentify run "task" starts fresh provider work by default. Agentify sends a compact repository-aware task prompt and does not ask the provider to continue prior history.

open world

agentify run opens a template provider interactively when no task is supplied. Non-interactive/custom launches require a task so the provider has a bounded objective.

load last

--continue resumes the provider's most recent conversation. --resume is the same alias for run; with sess, resume means Agentify session context instead.

route scout

--with-context, non-interactive runs, routed mode, and --explain-plan ask Agentify to build a managed execution plan before the provider command is constructed.

loadouts

Choose the mode that matches the fight.

The flags change provider launch behavior before a model sees the task. Pick the smallest loadout that gives the provider the right starting state.

fresh task

Normal bounded work

Default choice for a new implementation, fix, review, or explanation. Agentify keeps the prompt compact unless a context flag asks for selected source.

agentify run --provider codex "add validation for empty search"
interactive

Provider UI launch

Template providers are interactive by default for run. Use --interactive when a supplied task should still land in the provider's UI.

agentify run --provider codex --interactive "fix auth bug"
resume

Continue provider history

Use only when the provider's own latest thread is intentionally part of the work. Without this flag, run treats the request as a fresh task.

agentify run --provider codex --continue
routed

Bounded retrieval route

Routed mode gives the provider a retrieval-oriented prompt with ranked summaries and fetch guidance instead of injecting source slices by default.

agentify run --context-mode routed "map checkout retry flow"
selected context

Inject the useful files

--with-context runs the planner and includes selected files, related tests, and memory in the launch prompt. It also injects source slices even when routed mode is selected.

agentify run --with-context "write tests for payment retries"
explain plan

Show the route first

--explain-plan prints the execution plan JSON before the provider starts, so the chosen context can be inspected without changing the task.

agentify run --explain-plan "harden order lookup errors"
caveman

Spend fewer output tokens

--caveman adds a terse-output preamble for run and sess. Technical accuracy and risky confirmations stay normal prose.

agentify run --caveman=ultra "summarize auth risks"
provider pin

Choose the agent host

--provider selects the template provider command. Codex, Claude, Gemini, and OpenCode each receive the prompt with their own interactive and resume argument shape.

agentify run --provider gemini "trace login failures"
context rules

Know when source enters the arena.

Agentify separates provider history from repository context. Resume flags control provider history. Context flags control what Agentify adds to the prompt before launch.

task fresh unless resume flag
mode compact or routed
memory automatic run recall
plan managed context only
source with-context or compact plan
provider template command
resume provider history flag
session use sess for durable state
Compact default A task run without context flags uses a minimal repository-aware prompt. The provider can read files as needed.
Selected source --with-context asks the planner to include ranked files, related tests, and memory in the provider prompt.
Routed retrieval --context-mode routed sends bounded summaries and fetch instructions. Source slices are omitted unless --with-context is also present.
Explain plan --explain-plan prints the plan JSON before launch, useful when checking what Agentify selected.
Provider resume --continue and --resume map to provider-specific resume arguments such as Codex resume --last.
Durable sessions Use agentify sess when you need Agentify-owned session IDs, manifests, transcripts, forks, or handoffs.

Flags and examples are backed by the CLI help, README command table, and run dispatcher behavior.
