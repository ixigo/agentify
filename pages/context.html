mental model

Context is a scouting loop.

Use agentify context inside routed work when the provider needs exact code but should not receive whole source files by default.

scan

agentify context search <term> reads the local index and returns ranked references across symbols, files, modules, and semantic surfaces.

open

agentify context fetch <path> returns exact source only when paired with --lines A:B or --symbol <name>.

summarize

agentify context compact --session <id> writes compacted session facts for the routed workflow instead of expanding the live prompt with raw files.

inspect

agentify context status --session <id> reports managed context size, run history count, context facts, and any prepared child session.

moves

Four moves cover the routed turn.

The command returns JSON so agents can quote exact paths, line numbers, and session metadata back into their next step.

search

Find the doors

Start with a term. Results are ranked refs, not source dumps, so the next fetch can target the smallest useful path or symbol.

agentify context search checkout
fetch lines

Open a bounded room

Line ranges use ascending A:B syntax. Large ranges are truncated at the command's 240-line cap.

agentify context fetch src/core/context.js --lines 120:180
fetch symbol

Jump to a named target

Symbol fetch uses the index to find a matching function, class, export, or display name in the requested file, then returns its bounded range.

agentify context fetch src/core/context.js --symbol fetchContext
session

Manage the campaign log

Compact and status require a validated session ID. They pair with agentify sess when work needs durable routed state.

agentify context status --session <session-id>

safety rails

The dungeon walls are intentional.

These limits are part of the feature, not missing power. They keep provider-routed context small, reviewable, and tied to the repository.

Repository-relative paths context fetch rejects absolute paths and ../ escapes before reading a file.
Required range selector Fetch must include either --lines A:B or --symbol <name>; a bare file path is not enough.
Ascending line syntax --lines accepts only numeric ranges where the end is greater than or equal to the start.
240-line maximum Line and symbol fetches are bounded to 240 lines and mark the JSON result as truncated when the requested range is larger.
Index-backed symbols --symbol needs the local index to locate the symbol range inside the target file.
No direct database route Routed provider prompts should use agentify context, not nested scan/doc/up/query commands or raw SQLite reads.

agentify query is for host-side index inspection. agentify context is the bounded retrieval route providers are told to use during routed work.
