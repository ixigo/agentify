mental model

Three moves cover the handoff loop.

Use plan before execution, handoff after a session has useful state, and exec only when you intentionally need a lower-level wrapper around a command you provide.

scout

agentify plan asks the index to rank modules, files, symbols, related tests, verification commands, changed files, and constraints for a task before you spend provider context.

quest log

agentify handoff writes handoff.md and handoff.json inside a session directory so another agent can continue from a structured local bundle.

raw tunnel

agentify exec runs the command after -- through Agentify's execution wrapper. It is the advanced escape hatch, not the normal entry point for provider work.

plan

Scout with ranked context.

agentify plan is a read-only preview of the context Agentify would select for a task. It is useful before agentify run, before splitting work across agents, or when you want to inspect why certain files are likely relevant.

purpose

Preview the route

The planner uses the repo index and task text to choose modules, file excerpts, symbols, tests, verification commands, changed files, and constraints.

agentify plan "add retry tests"
explain

Show score reasons

Use --explain when you want the score breakdown in a readable format. Without it, the command prints JSON by default.

agentify plan --explain "fix login timeout"
handoff input

Feed the run route

Use the selected files and verification commands as the scouting report for agentify run or agentify run --with-context.

agentify run --with-context "fix login timeout"
caveat

Index required

If the index is missing or unreadable, rebuild it with agentify scan or agentify up before using planning, query, or context commands.

agentify scan
handoff

Pass the next agent a quest log.

agentify handoff operates on session artifacts. Point it at a session ID when you know the workstream, or let it use the latest saved session when one already exists.

purpose

Write a bundle

The command writes markdown and JSON bundle paths under .agentify/session/<id>/, including task, context summary, touched files, risks, and recommended tests.

agentify handoff --session <id> "next task"
session id

Resolve the workstream

You can pass --session <id> or place the ID after handoff. If neither is present, Agentify uses the latest session and errors when no sessions exist.

agentify handoff <id> "continue review"
json

Machine-readable output

Use --json to print the bundle payload and the generated markdown and JSON paths instead of the shorter success message.

agentify handoff --session <id> --json
caveat

Needs session state

Handoff is for saved sessions. Start or resume a durable workstream with agentify sess before relying on it for cross-agent continuity.

agentify sess run --name "checkout-retries"
exec

Drop to the raw command tunnel.

agentify exec is the lower-level wrapper for commands you provide yourself. It keeps Agentify's execution handling around a custom argv, but it does not choose a provider template for you.

purpose

Wrap a custom command

Use it when you need to run a specific provider or local command through Agentify's execution path instead of the normal agentify run template.

agentify exec -- codex exec "fix auth bug"
required

Arguments after --

The command requires argv after --. Without that separator and command, Agentify fails with an explicit usage error.

agentify exec -- <command...>
normal route

Prefer run for provider work

If you want Agentify to build a prompt and provider command from config, use agentify run. Keep exec for custom tunnels.

agentify run "fix auth bug"
caveat

You own the argv

Because the command is supplied directly, flags and quoting belong to the wrapped command. Agentify only receives the parsed argv after --.

agentify exec -- node scripts/task.js
route map

Where each command sits in the flow.

Planning starts before a provider launch. Handoff depends on saved session state. Exec bypasses the provider template and runs exactly the command after --.
