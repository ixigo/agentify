agentify risk

Run the danger board before the PR.

Risk turns the current diff into a blast-radius score, a list of likely impacted code, and a short regression-test queue. Treat it like a mission meter: it points at danger, but it does not guarantee safety.

score range 0..100 source .agentify/index.db modes worktree diff or --since
mission check blast radius 
# Score the current worktree diff
agentify risk

# Score everything changed since the PR base
agentify risk --since origin/main

# Send the full report to automation
agentify risk --since origin/main --json
mission input

Pick the board that matches the review moment.

The command always needs an Agentify index. Run it after agentify scan or agentify up so files, modules, symbols, imports, and test commands are available.

local edit check

Current worktree diff

Use the default mode while you are still editing. Agentify reads the changed files in the current checkout and scores the blast radius against the indexed graph.

agentify risk
pr base check

Since a commit or ref

Use --since <commit> for a branch review. It scores files changed since the chosen ref, such as the PR base branch or a known release point.

agentify risk --since origin/main
automation

Machine-readable output

Add --json when a script, CI job, or follow-up agent needs every field instead of the compact terminal report.

agentify risk --since origin/main --json
follow the trail

Inspect with query

When the report names an unexpected module, file, or symbol, use the query command page to inspect owners, dependencies, definitions, references, callers, and impact paths.

agentify query impacts --file src/auth/useAuth.ts
score model

The meter is bounded, indexed, and explainable.

Risk combines changed-file contribution, dependency fan-out, semantic relationships, impacted files, and impacted modules into a score capped at 100.

High

The diff likely touches shared code, fan-out, or multiple modules. Run the recommended tests and inspect the impact path before review.

report sections

Read the board from top to fallback.

The human report is intentionally short. The JSON report keeps the same sections with the full file, module, symbol, and test payloads.

score

Risk: <score>/100 (<level>) is the headline. Reasons call out signals such as the highest changed-file contribution, dependency fan-out, semantic centrality, module count, or semantic surfaces.

changed

Changed files include the path, git status, owning module when known, and per-file scoring details in JSON. The terminal view lists each file and its rounded contribution.

impact

Impacted modules, files, and symbols come from the indexed import graph plus semantic TypeScript/JavaScript facts. The graph only covers what Agentify indexed.

tests

Prioritized test commands are ranked from indexed package scripts and test-file relationships. A high priority means "run this first", not "run only this".

json

--json returns schema_version, changed_files, risk, impacted, prioritized_test_commands, and notes for scripts or agents.

coverage warning

No indexed test does not mean safe.

If Agentify cannot match an indexed test command to the impacted modules, the report emits a note instead of pretending the mission is clear.

possible note

No indexed test command covers the impacted modules.

That note means the index did not find a test command for the impacted area. Rebuild the index after adding package test scripts, then rerun risk and choose any missing manual checks yourself.
