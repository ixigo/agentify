Pack the right loadout before the run.

Skills are opt-in playbooks copied into the provider skill roots you choose. Browse the catalog to pick a play style, then use the CLI to install that skill into project or user scope.

install flow browse catalog
alias agentify skills providers codex · claude · gemini · opencode scopes project · user
loadout ready opt in 
# See every built-in skill and alias
agentify skill list

# Install every built-in skill into this repo for Codex
agentify skill install all --provider codex --scope project

# Install one skill for another provider
agentify skill install grill-me --provider claude --scope project
mental model

The catalog is the armory; the command equips the gear.

The standalone catalog explains when and why each skill helps. The CLI makes the selected skill usable by copying its bundled files from Agentify into the right provider directory.

browse

Use skill-catalog.html when choosing what a skill does, what workflow it supports, and which local prerequisites it expects.

inspect

agentify skill list prints the built-in inventory from Agentify metadata, including supported providers and aliases such as god-mode for worktree-autopilot.

equip

agentify skill install <name|all> copies skill directories into provider-owned skill roots. Existing targets are skipped unless --force is set.

refresh

agentify sync refreshes project-scope built-in skills only for provider skill roots that already exist in the repo, so upgrades keep opted-in loadouts current.

moves

Two subcommands cover the install loop.

Start with the inventory, then install either one named skill or the full built-in set. The plural command name works too: agentify skills list.

inventory

List built-ins

Shows the canonical skill names, installable providers, aliases, and short descriptions. Add --json when another tool needs the list.

agentify skill list
single item

Install one skill

Installs one skill by canonical name or alias. If an alias is used, the CLI resolves it and installs the canonical skill directory.

agentify skill install grill-me --provider claude --scope project
full set

Install all skills

Copies every built-in skill into the selected provider roots. This is the fastest way to make a repo ready for broad agent workflows.

agentify skill install all --provider codex --scope project
alias route

Use aliases safely

Aliases are normalized like skill names. For example, god-mode resolves to worktree-autopilot; gh-issue-autopilot resolves to gh-autopilot.

agentify skill install god-mode --provider all --scope project
choices

Provider and scope decide where the loadout lands.

The skill command intentionally separates the agent host from the install scope. Pick the provider that will read the skill, then decide whether the files belong to this repo or your user profile.

provider

One provider

Use --provider codex, claude, gemini, or opencode. If omitted, Agentify uses the current default provider when it supports skill installs.

agentify skill install caveman --provider codex
provider list

Multiple providers

skill install accepts comma-separated providers and all. The local provider is not installable because it has no skill root.

agentify skill install qa --provider codex,claude
project scope

Share with the repo

--scope project writes provider skill folders under the repository, such as .codex/skills/ or .claude/skills/.

agentify skill install tdd --scope project
user scope

Keep it personal

--scope user writes to your provider profile, such as ~/.codex/skills/, ~/.claude/skills/, ~/.gemini/skills/, or OpenCode config skills.

agentify skill install commit-creator --scope user
controls

Preview, replace, or export the result.

The shared CLI flags still matter here: use dry runs before changing files, force when refreshing an existing target, and JSON when feeding automation.

dry run show would_install / would_replace
skip existing skill without force
force replace existing target
json machine-readable result
