agentify issue-killer

Turn labelled issues into a supervised mission board.

Issue-killer pulls opted-in GitHub issues, gives each one an isolated Worktrunk worktree, and launches a watched tmux pane with the selected provider. The human stays in the control room; agents only get the issues selected for this run.

# Inspect the selected queue first
agentify issue-killer --label agentify-ready --agent-provider codex --limit 5 --dry-run

# Launch supervised worktrees and tmux panes

tmux attach -t gh-issue-killer
mental model

Issues move through a gated queue.

The command is built for supervised parallel work. It does not scan the whole backlog by default; it starts from an explicit issue URL list or the opt-in label you choose.

queue

With no explicit URLs, Agentify queries open GitHub issues matching --label. The default label is agentify-ready, so maintainers decide what can enter the board.

worktree

Each selected issue gets a branch such as issue/<number>-<slug> and a Worktrunk worktree. Existing worktrees for the same branch are reused.

pane

Each worktree launches in tmux with a prompt that pins the assigned issue, branch, path, verification, commit, push, and draft PR responsibilities.

supervise

The summary prints the tmux session, attach command, issue URLs, branch names, and worktree paths so the human can watch and steer every pane.

preflight first

Prerequisites and safety controls come before launch.

A live run creates worktrees and starts provider CLIs. Check this board before moving from dry-run to launch.

required

GitHub access

The command requires a Git checkout, the GitHub CLI, and an authenticated GitHub session. Run gh auth status when setup is uncertain.

gh auth status
live only

Worktree control

Non-dry-run launches also require wt for Worktrunk worktree creation and tmux for supervised pane orchestration.

command -v wt tmux
agents

Provider CLIs

Agentify v1 supports codex and claude as agent providers for issue-killer panes. The selected CLI must be available for live runs.

agentify issue-killer --agent-provider codex --dry-run
selection

Opt-in queue

Use the default agentify-ready label, a custom --label, or explicit --issue-url values. Empty labels are rejected unless explicit URLs are supplied.

agentify issue-killer --label agentify-ready --dry-run

Dry-run still validates GitHub access and issue selection, but it does not create worktrees or launch tmux panes.

issue-killer source session command
guardrails

The board is capped and permission bypass is explicit.

The defaults are conservative for supervised v1 runs. Raise scope deliberately and keep provider permission bypass off unless the run has been approved for that mode.

Default limit: 5 Label-based runs select up to five open issues unless --limit is set.
Maximum limit: 10 --limit must be a positive integer and cannot exceed ten supervised panes.
Partial runs are opt-in If fewer open issues are found than requested, pass --allow-partial to launch fewer panes.
Explicit URLs are exact --issue-url and --issue-urls select only those GitHub issue URLs and default the limit to the URL count.
GitHub only in v1 --issue-provider github is supported. GitLab and Azure DevOps are rejected as future providers.
Bypass is not default --bypass-permissions must be supplied explicitly. Without it, panes keep provider permission prompts and sandbox approvals.
launch examples

Start with dry-run, then launch the selected board.

These examples come after the safety controls because they create or preview supervised work. The bypass example is intentionally last.

preview

Inspect the default queue

Verify labelled issue selection and output shape without creating worktrees or tmux panes.

agentify issue-killer --label agentify-ready --agent-provider codex --limit 5 --dry-run
default board

Launch Codex panes

Launch up to five agentify-ready issues into the default gh-issue-killer tmux session.

agentify issue-killer --label agentify-ready --agent-provider codex --limit 5
custom scope

Use a smaller limit

Keep the board narrow when you only want one or two supervised panes in flight.

agentify issue-killer --label docs-ready --agent-provider claude --limit 2
exact issues

Target explicit URLs

Skip label selection and launch only the issue URLs supplied to this run.

agentify issue-killer --issue-url https://github.com/ixigo/agentify/issues/209 --agent-provider codex --dry-run
session

Reuse a control room

Choose a named tmux session, or pass --reuse-session when continuing an existing supervised board.

agentify issue-killer --label agentify-ready --session-name gh-docs-board --reuse-session
explicit bypass

Only when approved

Bypass mode changes provider launch permissions for every pane in this run. Leave it off for normal supervised work.

agentify issue-killer --label agentify-ready --agent-provider codex --bypass-permissions
control room

What appears after a live launch.

The command prints enough coordinates to inspect every assignment. Attach to tmux, read pane prompts, and keep issue-scoped work inside the generated branch and worktree.
