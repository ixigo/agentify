mental model

Five tools, one maintenance loop.

Treat these commands like workshop mechanics. Check the room, preview the sweep, then run only the cleanup or refresh that matches the state you saw.

guards

agentify hooks manages repo-local Git hooks. The managed pre-commit hook runs agentify check --hook, and the post-merge hook refreshes scan/doc state with the local provider.

map data

agentify semantic refresh rebuilds TypeScript/JavaScript semantic facts for configured projects so query, planner, and docs output have fresher symbol surfaces.

inventory

agentify clean prunes stale generated artifacts, old run outputs, invalid session folders, optional AFK artifacts, empty Agentify folders, and expired cache blobs.

cache bin

agentify cache inspects or garbage-collects the content cache under .agentify/cache/ without deleting the full index or documentation set.

reserved

agentify memory compress is intentionally a placeholder. It returns not_implemented and points to the future caveman-compress skill instead of compressing files today.

toolshed

Use the right tool for the right room.

Each command family has a small supported surface. Unsupported subcommands fail fast, so scripts should call the exact routes shown here.

guards

hooks

Install, inspect, or remove Agentify-managed Git hooks. Use this when contributors should get automatic local guardrails around commits and merges.

  • install writes enabled managed hooks into .git/hooks/.
  • status reports managed preCommit and postMerge state.
  • remove strips only Agentify-managed hook blocks.
agentify hooks status agentify hooks install agentify hooks remove
map upgrade

semantic

Refresh semantic TS/JS project facts after source or configuration changes. The command supports only the refresh route.

  • refresh rebuilds semantic snapshots and updates indexed facts.
  • doctor --semantic is the inspection companion for stale or failing semantic projects.
  • Any other semantic subcommand is rejected by the dispatcher.
agentify semantic refresh agentify doctor --semantic agentify doctor --semantic --json --fail-on-stale
sweep preview

clean

Prune generated artifacts that Agentify can classify as stale or invalid. Preview first, then widen the cleanup with optional groups only when the report matches your intent.

  • --dry-run reports paths and cache blob counts without deleting them.
  • --planned includes AFK plan markdown under .agentify/planned/.
  • --sessions includes AFK session folders; --all includes both optional groups.
agentify clean --dry-run agentify clean --planned --sessions --dry-run agentify clean --planned --sessions
inventory bin

cache

Inspect cache size before garbage collection. Use cache GC when you want to trim blobs without running the broader cleanup path.

  • status prints blob count and total size, or JSON with --json.
  • gc removes unreferenced blobs after the configured max age.
  • --max-age overrides the default cache age window for one GC run.
agentify cache status agentify cache status --json agentify cache gc --max-age 14
reserved slot

memory

The memory helper is present so future compression can claim a stable command shape. Today it is not a working compression flow.

  • compress requires a file path argument.
  • Text output says compression is reserved for the caveman-compress follow-up.
  • JSON output reports status: "not_implemented".
agentify memory compress AGENTIFY.md agentify memory compress AGENTIFY.md --json agentify skill install caveman-compress
safety notes

Scout, then sweep.

The maintenance surface is intentionally small, but several commands touch local generated files. These checkpoints keep cleanup predictable.

first move

Preview cleanup

Run agentify clean --dry-run before destructive cleanup. The report includes removed paths, cache blob counts, and skipped categories such as missing indexes.

cache move

Inspect before GC

Use agentify cache status before cache gc. GC is narrower than clean, but it still removes old unreferenced blobs and updates the cache manifest.

hook move

Managed blocks only

hooks remove targets Agentify-marked hook blocks. Existing unmanaged hook content is preserved when Agentify can split the managed section cleanly.

reserved move

Do not script compression yet

memory compress is useful as a compatibility signal, not as a data transformer. Treat not_implemented as the expected current result.
