mental model

Query is the host-side inspection kit.

Run agentify query when you want deterministic answers from the local index without launching a provider. The command reads indexed structural tables, semantic metadata, and git change data, then prints JSON that scripts and agents can consume.

map

owner, deps, and changed answer module-level questions from the structural index plus git state.

scan

search combines structural index search with semantic index search for a term you provide.

trace

def, refs, and callers use semantic symbol facts to navigate definitions and incoming references.

radius

impacts walks internal semantic edges from a changed file and respects --depth when you need a bounded blast-radius check.

scanner deck

Eight scanners, one index.

These are the complete supported query subcommands in the CLI. Each scanner shows the purpose, required input, and a realistic command shape.

owner locator

Find the owning module

Answers which indexed module owns a file and includes semantic file context when available.

Input: --file <path>

agentify query owner --file src/core/provider.js
dependency radar

Inspect module dependencies

Returns structural and semantic dependencies for one indexed module, including inbound usage.

Input: --module <id>

agentify query deps --module core/provider
changed-map

Map changed files to modules

Compares git changes since a commit and groups changed files by indexed owning module.

Input: --since <commit>

agentify query changed --since origin/main
search scanner

Search indexed metadata

Searches structural index records and semantic context for a term. Use this for repo-index inspection.

Input: --term <value> or a positional term

agentify query search --term auth
definition marker

Resolve symbol definitions

Finds semantic definitions for a symbol and marks ambiguous matches when multiple candidates exist.

Input: --symbol <name>

agentify query def --symbol querySearch
reference tracker

List symbol references

Returns ranked semantic references that point to the resolved symbol definitions.

Input: --symbol <name>

agentify query refs --symbol querySearch
caller trace

Find callers and renderers

Filters semantic references down to incoming call and render edges so you can see who invokes a symbol.

Input: --symbol <name>

agentify query callers --symbol runAgentify
impact radius

Trace impacted files

Walks internal semantic edges from a file to rank files that may be affected by a change.

Input: --file <path>, optional --depth <n>

agentify query impacts --file src/core/query.js --depth 2
query vs context

Two search moves answer different questions.

query search is for inspecting what the repo index knows. context search is for bounded retrieval in routed provider workflows.

query search

Use agentify query search --term auth when you are in the host shell and want JSON from indexed structural and semantic search stores.

context search

Use agentify context search auth when a routed provider prompt needs a bounded set of retrievable context hits instead of a broad index-inspection report.

rule

If your next step is scripting, auditing, or deciding which graph command to run, use query. If your next step is feeding source slices to an agent, use context.

The context command page is tracked separately. Until it exists, compare behavior with the README and the context command source.

source map

What each scanner reads.

The command is index-backed, so stale results usually mean the repository needs an index refresh before querying again.

Structural index owner, deps, and search read module, dependency, and indexed metadata tables.
Semantic index search, def, refs, callers, and impacts use semantic file, symbol, reference, and edge facts.
Git state changed reads files changed since --since, then maps them to indexed modules.
Index guidance If the index is missing, query errors include guidance to run Agentify indexing before retrying.
JSON output Every scanner prints structured JSON, making the command useful for scripts and provider-readable diagnostics.
