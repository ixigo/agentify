Install once. Your coding agent uses it automatically.

Agentify gives AI coding agents lightweight, persistent context. Run agentify install once per repo — from then on the agent drives Agentify, not you.

Quick start GitHub Skill catalog

How it works

One command wires Agentify into your agent's configuration, then stays out of your way.

agentify install
  ├── CLAUDE.md             ← managed guidance block: how the agent should use agentify
  ├── .claude/settings.json ← Claude Code hooks:
  │     SessionStart → agentify ctx load   (inject context digest)
  │     PostToolUse  → agentify ctx track  (record edits + commands)
  │     SessionEnd   → agentify ctx track  (close out the session)
  └── .agentify/            ← lightweight JSONL context store + structural index
01

You ask for something — each prompt is matched against the store and only related notes and files are injected (set context.injection: digest for the old always-on digest).

02

Agent works — file edits and shell commands are tracked automatically as compact JSONL (auto-compacted, capped at ~512 KB).

03

Agent learns something worth keeping — it runs agentify ctx note "..." so the knowledge survives the session.

04

Session ends — a fast model compresses it into a ~3-line handoff for future sessions; agentify ctx share makes notes committable team memory.

No daemon, no database server, no per-command wrapping. Install and uninstall are surgical: only the marker-delimited block and Agentify's own hook entries are ever touched.

Quick start

# install the CLI straight from GitHub
curl -fsSL https://raw.githubusercontent.com/ixigo/agentify/main/install.sh | bash
# (equivalent: npm install -g github:ixigo/agentify)

# wire up a repo (CLAUDE.md + Claude Code hooks)
cd /path/to/your/repo
agentify install

# also works with Codex (writes AGENTS.md guidance)
agentify install --provider codex
agentify install --provider all

# or one global setup instead of per-repo files
agentify install --global

# optional: build the structural index for query/risk
agentify scan

# shell work out to the right model
agentify delegate quick "rename getUser to fetchUser" --write
agentify delegate review --diff origin/main

# check or undo at any time
agentify status
agentify uninstall

Works with

Claude Code automatic

Hooks track every file edit and command with zero agent effort. A SessionStart hook injects the context digest into each new session. Guidance lives in CLAUDE.md.

Codex guidance-driven

Codex has no lifecycle hooks, so agentify install --provider codex writes an AGENTS.md block teaching Codex to load context at session start and record notes as it works.

Any other agent CLI

Everything is a plain CLI with --json output: ctx load, ctx note, query, risk. Add equivalent guidance to your agent's instruction file.

Platform workflows

On GitHub, GitLab, or Azure DevOps — there's a prebuilt workflow to get things done. agentify workflow install detects your platform from the git remote and installs the bundle: triage the board, pick up an item, implement in an isolated worktree, raise a draft PR.

GitHub gh

github-triage label state machine, gh-autopilot execution, issue-killer parallel worktree fan-out, draft PRs via pr-creator.

GitLab glab

gitlab-triage, glab-autopilot for issues and MRs, the same worktree isolation and parallel fan-out, draft MRs.

Azure DevOps az

azure-devops-triage over Azure Boards, ado-autopilot for work items and Repos PRs, plus pr-convention-learner for reviewer conventions.

agentify workflow install   # auto-detects gh / glab / azure from the git remote
agentify workflow list      # bundles + platform CLI availability

You never invoke the workflow by name — the skills match plain requests. A typical day:

"triage the new issues"          → github-triage labels the board, marks agentify-ready
"pick up issue 231"              → isolated worktree, implement, test, cross-vendor review, draft PR
"kill everything agentify-ready" → issue-killer fans out parallel tmux worktree agents
next session                     → the SessionStart digest recalls what's in flight

Model routing

Install also sets up a routing table: each kind of work goes to the model best suited for it, and the agent is taught to delegate instead of doing everything inline.

KindDefault routeUsed for
quickClaude HaikuSmall, low-impact edits, mechanical changes, quick questions
implementClaude SonnetStandard feature work and multi-file refactors
heavyClaude OpusArchitecture decisions, deep debugging, high-risk changes
reviewCodexIndependent post-change review by a different vendor
researchClaude HaikuFast exploration, summarization, doc lookups
# after finishing a change, the agent gets a second opinion from another vendor
agentify delegate review --diff origin/main

**Findings**
- **High**: math.js:1 changes add(a, b) from addition to subtraction.
  Callers expecting add(2, 1) === 3 now get 1 — a direct behavioral regression.

Routes use version-independent model aliases so they don't rot, fall back across vendors when a CLI is missing, run read-only unless --write is passed, and are fully overridable in .agentify.yaml under models.routes.

What the agent sees

Context arrives when it's relevant. Ask about payment retries, and only the related notes and files are injected — a CSS question pulls nothing:

## Agentify context (relevant to this task)

### Related notes from earlier sessions
- [2026-07-05] payment retries: idempotency key lives in src/pay/retry.ts,
  do not regenerate per attempt

### Files previously worked on that look related
- src/pay/retry.ts (14 edits)
- src/pay/retry.test.ts (9 edits)

Full history: `agentify ctx load`.

Events are single-line JSON records under .agentify/context/. Command text is clipped, output is never captured, and secrets are redacted from run reports.

Commands

All commands accept --json — which is how agents call them.

CommandWhat it does
agentify install [--global] [--provider claude|codex|all]Wire Agentify into the repo or your home config
agentify uninstallRemove the managed block and hooks — nothing else
agentify statusIntegration + context-tracking status
agentify ctx loadDigest of recent activity, notes, hot files
agentify ctx note "<text>"Record a note for future sessions
agentify ctx handoff ["task"]Write a handoff summary
agentify delegate <kind> ["task"]Shell a task out to the routed model (--diff, --write)
agentify modelsModel routing table + provider availability
agentify scanBuild the SQLite structural index
agentify query …owner, deps, changed, search, def, refs, callers, impacts
agentify risk --since <ref>Blast radius + suggested regression tests
agentify up / agentify checkRefresh the index / validate freshness
agentify skill list|installInstall bundled agent skills (browse the catalog, command reference)
agentify workflow list|installPlatform workflows: GitHub, GitLab, Azure DevOps
agentify hooks installOptional git hooks (pre-commit check, post-merge rescan)
agentify doctor / clean / completionToolchain readiness, artifact pruning, shell completion

Design principles

Lightweight

Context is plain JSONL. The event log auto-compacts past ~512 KB. Hook commands are designed to never fail and never block the agent.

Reversible

Everything install writes is marker-delimited or prefix-identified. uninstall removes exactly what was added; your own config survives.

Agent-first

JSON output everywhere, structural queries over a deterministic index, and guidance blocks that teach the agent when to reach for each command.
